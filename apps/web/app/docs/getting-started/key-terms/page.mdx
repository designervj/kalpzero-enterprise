# Key terms

Before you go deeper into the docs, it helps to agree on the main words used in Kalp. This page keeps those words simple.

<h2 id="business-structure">Business structure</h2>

| Term | Meaning |
| --- | --- |
| Agency | The parent group that can manage one or more businesses |
| Business | One hotel, store, travel company, or other customer inside Kalp |
| Tenant | The internal system word for a business |

<h2 id="people">People</h2>

| Term | Meaning |
| --- | --- |
| Platform admin | A person who can create agencies, create businesses, and manage the overall system |
| Tenant admin | A person who manages one business |
| Public visitor | A person viewing a business website without using the dashboard |

<h2 id="settings-and-content">Settings and content</h2>

| Term | Meaning |
| --- | --- |
| Business settings | The saved rules that change a business's modules, menus, labels, and theme |
| Business Blueprint | The internal code name for the main business settings document |
| Page data | The saved content for public website pages |
| Discovery content | Summary content, cards, and listings used for search and discovery pages |
| Preview | A way to inspect changes before they are shown publicly |

<h2 id="deployment-terms">Deployment terms</h2>

| Term | Meaning |
| --- | --- |
| Vertical pack | The business type or product pack, such as hotel, commerce, or travel |
| Feature flag | A switch that turns a product feature on or off for a business |
| Website deployment | The repo and live website details returned when Kalp creates a website for a business |
